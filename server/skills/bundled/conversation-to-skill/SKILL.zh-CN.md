# 对话转技能

该技能用于把当前对话中形成的工作方式转化为一个可复用的 agent skill。

它的目标不是简单生成 `SKILL.md`，而是识别真正稳定、可迁移的工作流，把一次性上下文和长期有价值的方法分离开，并决定技能应放在用户级目录还是项目级目录中。

## 使用场景

当用户希望完成以下事情时使用：

- 把当前任务或工作流变成可复用技能。
- 保存一次成功协作模式，供后续会话使用。
- 标准化某类任务的处理方式。
- 从当前线程中抽取可重复的 agent 工作流。
- 把推理框架、执行顺序或产物模式打包成技能。
- 升级已有草稿技能，让它更通用、更容易触发、更容易执行。

## 核心流程

1. 通读当前对话，识别稳定模式，而不是照抄聊天记录。
2. 区分长期流程、一次性细节、用户偏好和环境约束。
3. 决定技能存放位置：
   - 用户级：适合跨项目复用。
   - 项目级：适合当前仓库或当前产品上下文。
4. 编写 `SKILL.md`，重点包含：
   - 清晰的触发描述。
   - 可执行的步骤。
   - 必要的约束、输入和输出。
   - 失败模式和验证方式。
5. 如果质量要求较高，补充轻量 eval、样例或检查脚本。

## 质量要求

- 不要只是复述当前对话。
- 不要把临时路径、一次性命令、偶发错误当成通用规则。
- 技能描述要足够具体，未来 agent 能判断何时应该使用它。
- 技能步骤要可执行，而不是抽象建议。
- 如果技能依赖脚本、参考资料或资产，应放在同一技能目录下保持自包含。

## 与 skill-creator 的关系

该技能可以借鉴 `skill-creator` 的方法：清晰描述、合理目录结构、可验证样例和反馈迭代。但它的重点是从真实对话中提取稳定流程。
