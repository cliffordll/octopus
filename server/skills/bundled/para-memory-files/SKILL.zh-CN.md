# PARA 文件记忆

这是一个持久化的文件记忆系统，使用 Tiago Forte 的 PARA 方法组织信息。所有路径都相对于 `$AGENT_HOME`。

## 三层记忆

### 第一层：知识图谱

路径：`$AGENT_HOME/life/`

使用 PARA 结构保存实体化知识。每个实体目录包含：

1. `summary.md`：快速上下文，优先读取。
2. `items.yaml`：原子事实，按需读取。

典型目录：

```text
$AGENT_HOME/life/
  projects/      # 有明确目标或期限的活跃工作
  areas/         # 持续责任领域
  resources/     # 参考资料和兴趣主题
  archives/      # 已归档内容
```

### 第二层：每日笔记

每日笔记保存原始时间线，用于记录当天发生的事件、观察、决策和待整理信息。

### 第三层：隐性知识

隐性知识用于记录用户偏好、协作模式、常见纠正、项目习惯和长期行为模式。

## 使用场景

- 保存一个事实或观察。
- 记录每日工作线索。
- 创建或更新实体。
- 回忆过去上下文。
- 维护计划文件。
- 执行每周综合整理。
- 处理记忆衰减和归档。

## 工作原则

- 事实应尽量原子化，避免把大段叙事塞进单个条目。
- `summary.md` 用于快速恢复上下文。
- `items.yaml` 用于结构化事实和证据。
- 每日笔记保留原始时间线，不必一开始就完全整理。
- 定期从每日笔记和原子事实中综合出更稳定的知识。
