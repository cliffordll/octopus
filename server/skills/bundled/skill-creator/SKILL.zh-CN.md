# 技能创建器

当任务是创建 skill、优化 skill，或判断一个 skill 是否应该修改时使用该技能。

## 使用场景

- 用户要求创建一个新的可复用技能。
- 用户希望把某个流程标准化。
- 用户要求改进已有 `SKILL.md`。
- 需要判断技能描述、触发条件或执行步骤是否有效。
- 需要把经验沉淀为未来 agent 可复用的能力。

## 核心原则

- 技能描述必须清楚说明何时使用。
- 技能步骤必须能指导未来 agent 执行，而不是只写抽象建议。
- 技能应聚焦一个明确能力，不要塞入多个不相关流程。
- 如果需要外部资料、脚本、模板或示例，应放入同一技能目录。
- 技能应尽量可验证，能通过样例、检查表或轻量 eval 判断是否有效。

## 创建流程

1. 明确技能目标和触发条件。
2. 确认技能使用者、输入、输出和限制。
3. 设计目录结构，决定是否需要 `references/`、`scripts/`、`assets/` 或 eval 文件。
4. 编写 `SKILL.md`：
   - frontmatter 中写清 `name` 和 `description`。
   - 正文中写清使用场景、步骤、约束和验证方式。
5. 检查描述是否足够具体，未来 agent 能否正确触发。
6. 根据反馈迭代。

## 常见问题

- 只描述目标，没有执行步骤。
- 触发描述太宽泛，导致不该用时也会触发。
- 把一次性项目细节写成通用规则。
- 缺少验证方式，无法判断技能是否有用。
