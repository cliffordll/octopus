# 消息侧栏会话列表设计

## 目标

将消息页二级导航精简为会话入口与会话列表，去掉不需要的筛选操作；聊天正文和发送区交互保持不变。

## 交互

- 保留 `+ 新建对话` 入口。
- 列出当前组织全部会话，继续显示标题、所选智能体名称与状态徽标。
- 移除 `搜索对话`、`状态` 与 `智能体` 三个侧栏筛选控件。
- 当组织尚无对话时展示 `暂无对话`。

## 边界

- 不修改聊天创建、消息发送或正文编辑器中的智能体选择功能。
- 不修改服务端接口或契约；继续读取会话列表和智能体列表以标注会话。

## 测试

- 侧栏中不存在三类筛选控件。
- 多状态、多智能体的会话都同时显示。
- 空列表显示 `暂无对话`。
