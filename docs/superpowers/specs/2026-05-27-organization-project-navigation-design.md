# 组织与项目导航分组设计

## 目标

将组织工作区二级侧栏拆分为 `组织` 与 `项目` 两组，并把项目详情组织为三级标签。实现范围仅覆盖当前服务端已有接口，不修改服务端代码。

## 导航结构

- `组织` 分组展示 `组织架构` 与 `心跳`。
- `项目` 分组通过 `GET /api/orgs/{orgId}/projects` 列出当前组织全部项目，点击后进入对应项目详情。
- 无项目时在项目分组展示空状态；项目列表页仍保留创建项目能力。
- 暂不展示没有组织级读取接口的资源、工作区、目标、技能、成本和活动。

## 视觉层级

- `组织架构`、`心跳` 与 `全部项目` 是侧栏主要入口，使用一致的主入口样式。
- 具体项目是 `全部项目` 下的次级条目，使用缩进、更小字号及较弱的常态文字颜色。
- 当前项目条目仍显示清晰选中态，并通过项目列表上方的分隔线强调其从属关系。

## 项目详情

- 项目详情增加三级标签：`配置`、`资源`、`任务`。
- `配置` 沿用项目描述及状态编辑能力。
- `资源` 沿用项目资源附件列表、添加和移除能力。
- `任务` 调用现有 `GET /api/orgs/{orgId}/issues?projectId={projectId}`，展示属于当前项目的任务并可进入任务详情。
- 由于当前没有项目预算接口，不展示 `预算` 标签。

## 测试范围

- 组织导航按两组显示，并列出服务端返回的项目链接。
- 组织主要入口与具体项目条目使用可区分的样式角色。
- 项目详情的三级标签具有稳定路由，配置与资源能力保持可用。
- 项目任务标签按 `projectId` 请求并呈现任务链接。
