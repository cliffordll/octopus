# 任务侧栏分组设计

## 目标

任务页二级导航改为分组结构，方便从任务视角、最近查看和项目维度进入任务。

## 导航结构

- `任务`
  - `全部任务`：展示组织全部任务。
  - `草稿任务`：使用现有 `backlog` 状态作为草稿入口。
  - `关注中`：服务端当前没有关注筛选字段，先保持与其他入口一致的导航样式，不发起不存在的筛选请求。
- `最近查看`
  - 打开任务详情页时在浏览器本地记录最近查看任务。
  - 二级导航读取本地记录并展示最近任务链接。
- `项目`
  - 拉取当前组织项目列表和全部任务。
  - 按 `projectId` 将任务展开到对应项目下。
  - 每条任务继续展示任务状态徽标。
  - 没有任务的项目只显示项目名称，不在二级菜单显示空态文案。

## 边界

- 不修改服务端代码和接口契约。
- 不新增服务端字段；没有服务端支持的关注功能只做占位。
- 保留任务列表页现有状态筛选和新建任务能力。

## 测试

- 任务导航显示 `任务`、`最近查看`、`项目` 三组。
- `全部任务`、`草稿任务`、`关注中` 都在 `任务` 分组中。
- 点击 `草稿任务` 使用 `status=backlog` 拉取任务。
- 项目分组按项目展示对应任务，并带状态徽标。
- 打开任务详情后会记录最近查看任务。
