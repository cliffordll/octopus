# 智能体详情操作入口设计

## 目标

将智能体详情页中的运行操作提升到页面顶部，与智能体名称处于同一操作区，并补齐分配任务和聊天入口。所有变更仅落在前端，复用服务端已经提供的任务创建与对话创建接口。

## 交互

- 顶部操作区展示 `分配任务`、`聊天`、`暂停`、`恢复`、`终止`、`运行心跳`。
- 智能体名称旁展示当前状态徽标，使配置与运行标签下仍能直接确认状态。
- 生命周期及 Heartbeat 操作在概览、配置、运行三个标签中都可用，不再重复出现在概览卡片内。
- `分配任务` 打开弹窗，当前智能体固定为负责人；用户录入标题后创建任务，成功后进入任务详情。
- `聊天` 进入新对话页面，并通过查询参数预选当前智能体。只有支持聊天且未终止的 `codex_local` 智能体可进入聊天。

## 边界

- 不修改 `server/` 或共享服务端代码。
- 不在弹窗增加当前接口未要求的高级任务字段。
- 生命周期按钮继续遵循现有状态禁用规则。
